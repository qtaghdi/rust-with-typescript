```typescript title="TypeScript"
// 에러가 타입에 없음
async function getUser(id: string) {
  const res = await fetch(`/api/users/${id}`);
  const user = await res.json(); // any 타입
  return user.name.toUpperCase(); // 런타임에 터질 수 있음
}
```

```rust title="Rust"
// 에러가 타입에 명시됨
async fn get_user(id: &str) -> Result<String, Error> {
  let user: User = fetch_user(id).await?; // 컴파일타임 검증
  Ok(user.name.to_uppercase()) // None이면 컴파일 에러
}
```

</div>

---

<Aside type="tip" title="이 가이드는 누구를 위한 건가요?">
TypeScript로 실무 개발을 하고 있고, Rust를 배워보고 싶은 분을 위해 쓰였습니다.
`any` 쓰는 걸 싫어하고, 타입이 주는 안전망의 가치를 알고 있다면 — Rust가 잘 맞을 겁니다.
</Aside>

## 학습 경로

<Steps>

1. **사고방식 전환** (Ch.1–2)

   문법 전에 철학부터. GC 없이 메모리가 안전한 이유, 컴파일타임이 런타임보다 더 많은 걸 보장하는 이유를 이해합니다.

2. **핵심 언어 습득** (Ch.3–5)

   TypeScript와 1:1로 대응하는 Rust 문법, **Ownership & Borrowing**, **Cargo & 모듈 시스템**을 차례로 소화합니다.

3. **타입 시스템 마스터** (Ch.6–9)

   Enum, 컬렉션, 이터레이터, Trait까지. 이 단계를 끝내면 Rust 코드를 읽고 쓰는 데 막히지 않습니다.

4. **실전 적용** (Ch.10–12)

   실제 프로젝트 패턴, 테스트 작성, 6개월 로드맵으로 독학 계획을 세웁니다.

</Steps>

## TypeScript → Rust, 뭐가 바뀌나요?

| | TypeScript | Rust |
|--|--|--|
| 타입 안전성 | 컴파일 타임 (부분적) | 컴파일 타임 (전면적) |
| Null 안전성 | `T \| null` + optional chaining | `Option<T>` + exhaustive match |
| 에러 처리 | `try/catch` (타입 없음) | `Result<T, E>` (타입 강제) |
| 메모리 | GC (자동, pause 있음) | Ownership (zero-cost) |
| 성능 | Node.js 수준 | C/C++ 수준 |
| 동시성 안전성 | 런타임에서만 확인 | 컴파일 타임에 데이터 레이스 차단 |

Rust는 TypeScript가 하고 싶었던 것의 **완성판**처럼 느껴질 수 있습니다.

<Aside type="caution" title="러닝 커브에 대해서">
Ownership과 Borrow Checker는 처음엔 컴파일러와 싸우는 느낌을 줍니다.
하지만 그 싸움에서 이기고 나면, **"이 코드는 안전하다"는 확신을 가지고 배포**할 수 있게 됩니다.
</Aside>
